  Project Notes

  opus4.8-cloud-claude is the strongest template-quality project. It has a clean src/ layout, strict mypy, ruff, no unnecessary runtime dependencies, fixture-backed providers, a fake/Ollama reasoner abstraction, CLI, storage, evaluation, and broad tests. It is less
  ambitious than Hermes, but much more disciplined.

  minimax-m3-cloud-hermes is the most feature-rich: FastAPI, CLI, SQLite, calibration, Elo, fixture generation, prompt sweeps, local/OpenAI-compatible LLM support, and a large eval harness. It is also visibly generated and accumulated: many lint errors, committed
  runtime artifacts/caches/databases, some import-path fragility, no mypy gate, and noisy code. Good prototype, weaker company-standard template.

  gpt5.5-cloud-codex is clean, small, and passes all quality gates. It has good separation around tools, storage, evaluation, and a World Cup catalog loader. Its weakness is scope: fewer tests, less evaluation depth, and less of a true LLM-agent implementation than
  Opus or Hermes.

  glx4.7-cloud-claude has the shape of an ambitious LangGraph/SQLAlchemy agent, but it is not healthy. Tests fail immediately, the evaluation workflow module appears to contain test code instead of the implementation, it ignores the expected src/ layout, and it
  carries heavy dependencies for an incomplete surface.

  gemma4-12bmlx-local-codex is effectively unusable. Core files contain syntax errors, misspelled imports/classes, broken packaging (setuptools.devrsytcs), non-discovered tests, and comments that look like unresolved model self-dialogue.

  Recommendation

  Use opus4.8-cloud-claude as the baseline if the goal is maintainable production-template quality. Borrow ideas from minimax-m3-cloud-hermes only selectively, especially its evaluation/calibration concepts, after cleaning them behind strict linting, typing, and
  artifact hygiene.
